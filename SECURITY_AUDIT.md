# Security Audit Report: Shark Agent Installation Scripts

## CRITICAL VULNERABILITIES

### [Unverified Script Execution via curl | bash]
- **Severity:** Critical
- **Location:** Both scripts - primary installation method
- **Description:** The recommended installation method is `curl | bash` with no integrity verification. Users are instructed to pipe arbitrary content from GitHub directly to bash without checksum verification, signature validation, or TLS certificate verification beyond what curl provides.
- **Exploit Scenario:** Attacker compromises GitHub account/repository, DNS hijacks raw.githubusercontent.com, or performs MITM attack to inject malicious code executed with user privileges.
- **Fix:** 
  ```bash
  # Add SHA256 checksum verification
  SCRIPT_URL="https://raw.githubusercontent.com/leviathan-devops/shark-agent/main/setup.sh"
  EXPECTED_SHA256="verified_checksum_here"
  TEMP_SCRIPT=$(mktemp)
  curl -fsSL "$SCRIPT_URL" > "$TEMP_SCRIPT"
  ACTUAL_SHA256=$(sha256sum "$TEMP_SCRIPT" | cut -d' ' -f1)
  if [ "$ACTUAL_SHA256" != "$EXPECTED_SHA256" ]; then
      echo "ERROR: Checksum verification failed" >&2
      exit 1
  fi
  bash "$TEMP_SCRIPT"
  rm "$TEMP_SCRIPT"
  ```

### [API Key Stored in Plaintext with Insufficient Permissions]
- **Severity:** Critical
- **Location:** setup.sh:145, install-skill.sh:68
- **Description:** API keys are stored in plaintext JSON files (~/.shark-agent/config.json) with default permissions (typically 644). The scripts don't set restrictive file permissions, allowing other users on the system to read the keys.
- **Exploit Scenario:** Multi-user system or compromised user account allows attacker to steal API key, incurring financial costs or accessing private data through the DeepSeek API.
- **Fix:**
  ```bash
  # After creating config.json
  chmod 600 "$CONFIG_DIR/config.json"
  
  # Ensure directory permissions are secure
  chmod 700 "$CONFIG_DIR"
  ```

### [API Key Echoed in Terminal During Input]
- **Severity:** High
- **Location:** setup.sh:141
- **Description:** `read -p "Enter your DeepSeek API key: " API_KEY` shows the API key as it's typed, potentially exposing it to shoulder surfing, screen recording, or terminal history.
- **Exploit Scenario:** User runs script in public space, attacker reads API key from screen. Terminal history may capture the key if shell is configured to record interactive input.
- **Fix:**
  ```bash
  read -sp "Enter your DeepSeek API key: " API_KEY
  echo  # Add newline after silent input
  ```

### [Unsafe NPM Package Installation with Arbitrary Fallbacks]
- **Severity:** High
- **Location:** setup.sh:80-89
- **Description:** Script attempts multiple npm package names (`@anthropics/qwen-code`, `qwen-code`, `@qwen-code/qwen-code`) without verification. If the intended package is unavailable, it falls back to cloning and linking arbitrary GitHub repositories.
- **Exploit Scenario:** Typosquatting attack - attacker publishes malicious package named `qwen-code`. When the legitimate package fails to install, script installs the malicious one. GitHub repository could be compromised.
- **Fix:**
  ```bash
  # Verify package signature or checksum
  npm install -g @anthropics/qwen-code || {
      echo "ERROR: Failed to install verified package" >&2
      exit 1
  }
  # Do NOT fall back to arbitrary repositories
  ```

### [Privilege Escalation via Uncontrolled sudo]
- **Severity:** High
- **Location:** setup.sh:47, 51, 55, 60, 84-89
- **Description:** Script executes `sudo` without validating the commands or checking if they're necessary. Packages are installed globally via sudo npm, which could be hijacked if npm config is compromised.
- **Exploit Scenario:** Malicious npm package with postinstall script executes arbitrary commands as root. User's npm config could be modified to use malicious registry.
- **Fix:**
  ```bash
  # Check if already installed first
  if ! command -v node &> /dev/null; then
      echo "Please install Node.js manually from https://nodejs.org" >&2
      exit 1
  fi
  
  # Install npm packages without sudo when possible
  npm config get prefix | grep -q "$HOME" || {
      echo "ERROR: npm prefix is not in user directory" >&2
      exit 1
  }
  npm install -g @anthropics/qwen-code
  ```

### [Insecure Temporary File Handling]
- **Severity:** High
- **Location:** setup.sh:91-96, 114-121; install-skill.sh:41-62
- **Description:** Scripts use predictable temporary paths (`/tmp/qwen-code`, `/tmp/shark-temp`) and don't clean up properly on error. Race conditions possible if multiple users run script simultaneously.
- **Exploit Scenario:** Attacker creates symlink at `/tmp/shark-temp` before script runs, causing files to be written to arbitrary locations. Or attacker reads temporary files containing sensitive data.
- **Fix:**
  ```bash
  TEMP_DIR=$(mktemp -d)
  trap "rm -rf '$TEMP_DIR'" EXIT ERR INT TERM
  
  git clone --depth 1 https://github.com/QwenLM/qwen-code.git "$TEMP_DIR/qwen-code"
  cd "$TEMP_DIR/qwen-code"
  # ... operations
  ```

### [Unsafe Input Handling in JSON Generation]
- **Severity:** Medium
- **Location:** setup.sh:145-156
- **Description:** API key is directly embedded into JSON without escaping, allowing JSON injection if key contains special characters like `"`, `\`, or newlines.
- **Exploit Scenario:** Malicious API key containing `"` could break JSON structure or inject additional fields. While unlikely from legitimate API keys, it's a vulnerability.
- **Fix:**
  ```bash
  # Use jq for safe JSON generation
  if command -v jq &> /dev/null; then
      echo "$API_KEY" | jq -R --arg model "deepseek-reasoner" \
          '{api_key: ., model: $model, timeout: 120, max_loops: 10, yolo_mode: true, verbose: false}' \
          > "$CONFIG_DIR/config.json"
  else
      # Manual escaping as fallback
      API_KEY_ESCAPED=$(printf '%s' "$API_KEY" | sed 's/["\\]/\\&/g')
      cat > "$CONFIG_DIR/config.json" << EOF
  {
    "api_key": "$API_KEY_ESCAPED",
    "model": "deepseek-reasoner",
    "timeout": 120,
    "max_loops": 10,
    "yolo_mode": true,
    "verbose": false
  }
  EOF
  fi
  ```

### [Unrestricted File Permissions on Downloaded Scripts]
- **Severity:** Medium
- **Location:** setup.sh:124; install-skill.sh:74-75
- **Description:** `chmod +x` is applied to all `.py` files in the skill directory without validation. If the repository is compromised, malicious scripts gain execution permission.
- **Exploit Scenario:** Attacker adds malicious Python file to repository that gets executed automatically or through other means.
- **Fix:**
  ```bash
  # Only chmod specific, expected files
  for file in "run.py" "shark-brain.py"; do
      if [ -f "$SKILL_DIR/$file" ]; then
          chmod 755 "$SKILL_DIR/$file"
      fi
  done
  ```

### [Insecure Environment Assumptions]
- **Severity:** Medium
- **Location:** Both scripts - multiple locations
- **Description:** Scripts assume `$HOME/.bash_aliases` exists and is sourced, don't handle other shells (zsh, fish), and don't verify if changes were actually applied.
- **Exploit Scenario:** User with different shell configuration ends up with non-functional installation or has to manually fix shell config.
- **Fix:**
  ```bash
  # Detect shell and add to appropriate config
  detect_shell_config() {
      case "$SHELL" in
          */bash) echo "$HOME/.bashrc" ;;
          */zsh) echo "$HOME/.zshrc" ;;
          */fish) echo "$HOME/.config/fish/config.fish" ;;
          *) echo "$HOME/.profile" ;;
      esac
  }
  CONFIG_FILE=$(detect_shell_config)
  ```

### [Supply Chain Attacks via Git Repository]
- **Severity:** Medium
- **Location:** setup.sh:114; install-skill.sh:41-62
- **Description:** Script clones GitHub repositories without verifying commits, tags, or signatures. Compromise of the GitHub account or repository leads to automatic installation of malicious code.
- **Exploit Scenario:** Developer account hacked, attacker pushes malicious code that gets automatically installed by all users.
- **Fix:**
  ```bash
  # Verify git commit signature (if available)
  git clone --depth 1 https://github.com/leviathan-devops/shark-agent.git "$TEMP_DIR"
  cd "$TEMP_DIR"
  if git verify-commit HEAD 2>/dev/null; then
      echo "Commit signature verified"
  else
      echo "WARNING: Commit not signed or signature verification failed"
      read -p "Continue anyway? [y/N] " -n 1 -r
      [[ $REPLY =~ ^[Yy]$ ]] || exit 1
  fi
  ```

### [Bashrc Sourcing in Non-Interactive Shell]
- **Severity:** Low
- **Location:** setup.sh:183; install-skill.sh:78
- **Description:** Script attempts to source `$BASH_ALIASES` even when running in a non-interactive shell (`curl | bash`), which may have unintended side effects or fail silently.
- **Exploit Scenario:** Script fails to apply aliases, confusing users. Or sourcing causes errors in non-interactive environment.
- **Fix:**
  ```bash
  echo "Please run 'source ~/.bash_aliases' or restart your shell to apply changes"
  # Do not attempt to source automatically
  ```

- **Exploit Scenario:** Other users on multi-user system can see API key via `ps aux` or `/proc` filesystem.
- **Fix:**

  ```bash
  # Already addressed by using read -s
  # Ensure no command exposes the key
  ```
